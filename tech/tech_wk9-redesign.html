Website Redesign aka Whack-a-Mole

August 21.2014

I recently dedicated a few days to redesigning this blog site. I had made significant changes probably five or six times already, but this time was a complete overhaul. I want to mention a few things about the process.

What Needed to Change

Previously, all of my blog posts for each category (i.e. tech, culture and projects) were on a single HTML page. I had made the posts collapsible to only show the headings, so that one page served as both a list of blog titles, and a place for the content of each. As I added to this page, I realized the HTML page was getting a too big and rather inflexible. I wanted to separate each post to its own page, but then I'd need an archive page. I looked back at my homepage and realized it was pretty unnecessary: it merely provided links to the other 3 pages, each of which did the same thing. So, I wanted to add some functionality to it. Finally, I needed to add a "contact" section somewhere.

Bootstrappin'

The guys at Twitter graciously provided a front-end design framework that has become quite popular among developers. The CSS file was over 6000 lines long. I decided to download the CSS and JS files and give it a shot. A bit of a lot to chew at first. To really understand the framework, you'd have read the lengthy documentation and experiment with it. So, I set up the Bootstrap CSS file in one window and my own HTML and CSS file in the other. As I thought of new design ideas I searched through the docs and CSS to see what tools were available to help me achieve my design goals.

The grid styling provided an excellent framework for setting page items out in an organized and responsive fashion. I ended up placing most of the elements on my site in one of the grid divs. But of course, I wanted to customize everything so it didn't look like it came straight out of the Bootstrap box. Doing so was a complex process. I had to sift through the Bootstrap CSS to figure out how each element behaved, and then override it as necessary in my own CSS file.

I ended up modifying almost everything that I did use from Bootstrap. One of the tools I changed minimally was the nav bar framework for the individual blog pages. Boostrap provided the CSS and JS for its nav bar to collapse for mobile into a drop-down menu that I found incredibly useful. Ultimately I treated Bootstrap as a toolbox more than a framework for my site. I utilized bits and pieces as I went along, but never let it dictate my design.

Challenges

One of the toughest design challenges I faced was the background image. I realize that a lot of sites do not have a full scrolling page background image, and I think maybe that is because it can be buggy and difficult to work in concert with the content. However, I wanted to give it a shot. The background image is actually fixed in the HTML body. The hazy section banners are separate background images with non-blurred content divs on top. The banner design was a relatively simple hack that I figured out after only seeing complex methods online. The whole combination makes it look like hazy bars scroll seamlessly over the constant background.

Another challenge was implementing the sticky header buttons. This was a feature I had seen on other sites and really wanted to utilize it on my own. The home buttons stick to a fixed position on the top of the page once you scroll far enough down. A little JavaScript magic made this possible. But then I had to make them transparent and small to not step on the content. You can probably see how this becomes an endless redesign process

What Still Needs to Change

I feel that I was mostly successful in separating my blog posts, making a home page that serves as an archive with blog summaries, and utilizing some cool features. In fact, I was very happy with my site until I uploaded it and viewed it on my Galaxy. Ugh. The background effects don't look like they did on the computer. The nav buttons are too big. The margins are too wide. And on the iPhone, the initial home page view is all stretched out. I can adjust the margins and button placement for media queries, but I may have to scrap the background image for mobile sanity.

A Learning Experience

I could have predicted some of these mobile issues by resizing my computer browser, but other things came as a surprise. The whole redesign process feels a lot like playing Whack-a-Mole. As soon as I fix one issue, another pops up. And, there's always something can be improved. By using Bootstrap, but not following it too closely, I was able to add some cool features to my site without sacrificing uniqueness. However, in the attempt to customize and experiment, I lost some functionality. In any event, I can definitely say I improved my technical understanding by drilling into the code.
